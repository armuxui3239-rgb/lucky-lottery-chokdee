# Global User Rules (Strict Professional Workflow v2)

The user has explicitly mandated the following strict rules for **EVERY TASK**. You MUST follow these exactly as written, without exception.

## 1. Requirement Analysis (การวิเคราะห์ความต้องการ)
- **อ่านและทำความเข้าใจ:** ต้องอ่านและทำความเข้าใจความต้องการอย่างละเอียดก่อนวิเคราะห์หรือวางแผนเสมอ
- **หาข้อมูลมาตรฐาน:** อ่านความต้องการเพื่อหาข้อมูลเกี่ยวกับสิ่งที่บีฟมา โดยการค้นหาข้อมูลจากแหล่งอ้างอิงที่เชื่อถือได้ตามมาตรฐานสากลเป็นสิ่งแรก
- **เข้าใจองค์ประกอบโปรเจค:** ต้องทำความเข้าใจองค์ประกอบของโปรเจคดั้งเดิมเป็นอย่างดี และวางแผนการสร้างตามลำดับความเป็นมืออาชีพ
- **หากไม่มั่นใจให้ถาม:** หากมีส่วนใดที่ไม่ชัดเจนหรือไม่มั่นใจ ให้ถามกลับผู้ใช้งานทันที ห้ามคาดเดาเอง
- **Antigravity Skills:** ในโปรเจกต์จะมีชุดทักษะที่เอาไว้ให้คุณเรียกใช้ในโฟลเดอร์ `Antigravity Skills` ต้องศึกษาและนำมาใช้ให้เกิดประโยชน์สูงสุด

## 2. Strict Workflow Protocol (ขั้นตอนการทำงานที่เคร่งครัด)
คุณต้องใช้ขั้นตอน 4 ขั้นตอนนี้ในทุกๆ งาน (Task) โดยไม่มีข้อยกเว้น:

1.  **แจ้ง (Inform):** บอกรายละเอียดและผลลัพธ์ที่คาดหวังของสิ่งที่จะทำก่อนเริ่ม
2.  **ตรวจสอบ (Review):** แสดงชุดคำสั่ง/การกระทำ และตรวจสอบโปรเจคอย่างละเอียด **ห้ามทำงานซ้ำซ้อน**
3.  **อนุมัติ (Approve):** ต้องรอคำว่า **"ใช่" (ทำ)** หรือ **"ไม่ใช่" (ไม่ทำ)** จากผู้ใช้งานก่อน
    - **กฎการรันอัตโนมัติ:** เมื่อผู้ใช้งานตอบว่า "ใช่" ไม่ว่าจะเป็นการรันคำสั่งหรือการยืนยันใดๆ **คุณต้องดำเนินการต่อด้วยตัวเองทันที**
    - **ห้ามแช่หรือเงียบ:** หากเกิดปัญหา ติดขัด หรือข้อมูลไม่ถูกต้อง ต้องแจ้งให้ทราบทันที ห้ามทิ้งจอนิ่งหรือเงียบไปโดยไม่มีการแจ้งสถานะ
4.  **รายงาน (Report):** สรุปผลการทำงานทั้งหมดหลังจากเสร็จสิ้นภารกิจ

## 3. Communication & Tools (การสื่อสารและเครื่องมือ)
- **ภาษา (Language):** ใช้ภาษาไทย (Thai) เป็นหลักในการสื่อสาร วิเคราะห์ และสรุปผล
- **เคร่งครัด (Strict):** ห้ามละเลยหรือข้ามขั้นตอนเหล่านี้เด็ดขาด
- **การใช้ทักษะอัตโนมัต (Auto-Equip Skills):**
    - **Equip Debug Skill:** ทดสอบฟีเจอร์, ระบุบั๊ก, เสนอแนวทางแก้ไข (ใช้ `debug-agent`, `systematic-debugging`)
    - **Equip UI Skill:** ออกแบบ Responsive UI ตาม Brand Design [อ้างอิงตามบีฟ] (ใช้ `ui-ux-pro-max`, `frontend-design`)
    - **Equip Brainstorm Skill:** นำเสนอ 10 ฟีเจอร์ใหม่พร้อมแนวทางการหารายได้ (ใช้ `brainstorming`)
    - **Equip Test Skill:** รันการทดสอบ E2E เต็มรูปแบบ รายงานผล (ใช้ `webapp-testing`, `qa-agent`)
    - **Equip SaaS Skill:** สร้าง Full SaaS (Auth, DB, Stripe, Deployment) (ใช้ `ai-full-product-builder`, `ai-saas-pipeline`)
